4. Data word to be transmitted by the sender is 1001101. Assuming the even parity, find out the code word that will be sent to the receiver using Hamming code.

Hamming Code (Even Parity)

Data Word: 1001101 (7 bits)

1. Determine Parity Bits (r)

Formula: $2^r \ge m + r + 1$

For m=7, r=4 (since $2^4 = 16 \ge 7+4+1=12$)

Need 4 Parity Bits

2. Bit Positions (Total 11 bits)

Parity at $2^n$ positions: 1, 2, 4, 8

Data ($D_1$ to $D_7$) fill others.

C11 C10 C9 C8 C7 C6 C5 C4 C3 C2 C1

D7 D6 D5 P8 D4 D3 D2 P4 D1 P2 P1

3. Calculate Parity Bits (Even Parity)

Data Word: 1001101

P1 (1,3,5,7,9,11): 1,0,1,1,1 (4 ones) $\implies P_1=0$

P2 (2,3,6,7,10,11): 1,0,1,0,1 (3 ones) $\implies P_2=1$

P4 (4,5,6,7): 0,0,1 (1 one) $\implies P_4=1$

P8 (8,9,10,11): 1,0,1 (2 ones) $\implies P_8=0$

4. Final Codeword

10101001110

5. Explain the different types of frames exchanged during data transmission in HDLC protocol.

HDLC Frame Types

Bit-oriented protocol using specific frames for data and control.

Information Frames (I-frames)

Transport user data; carry flow/error control (piggybacking).

Includes $N_S$ (sent) and $N_R$ (received) sequence numbers.

Supervisory Frames (S-frames)

Flow and error control (ACKs, NAKs) when no user data.

Types: RR, RNR, REJ, SREJ.

Unnumbered Frames (U-frames)

Link management (setup/teardown, error reporting not data related).

No sequence numbers. e.g., SABM, DISC, UA.

Section C

1. Explain distance vector routing algorithm with an example.

Distance Vector Routing Algorithm

Routers exchange "distance vectors" to find shortest paths (Bellman-Ford).

Core Principles

  • Knows only neighbors.
  • Exchanges full DV with neighbors.
  • Updates via Cost(Dest) = min[Cost(Neighbor) + Cost(N to Dest)].
  • "Routing by Rumor."

Example (Simplified)

Router A DV (Initial): {B:1, D:5}

Router B DV (Initial): {A:1, C:1}

A learns C via B:

A to C = A-B (1) + B-C (1) = 2

A's Updated DV: {B:1, C:2, D:5}

Problems

  • Count-to-Infinity (routing loops).
  • Slow Convergence.

3. Discuss in detail the design of Fiber Optic cables and how it transmits data? Explain its types.

Fiber Optic Cables

Transmits data as light pulses over glass/plastic fibers.

Cable Design

  • Core: Pure glass/plastic, high refractive index (light travels).
  • Cladding: Lower refractive index (reflects light back via TIR).
  • Buffer Coating: Plastic protection.
  • Strength Members (Kevlar), Outer Jacket.

How Data Transmits (TIR)

  • Electrical to Optical (LED/Laser).
  • Light injected into core.
  • Total Internal Reflection (TIR): Light bounces off core-cladding boundary.
  • Light propagates with minimal loss.
  • Optical to Electrical (Photodiode).

Types of Fiber

Single-Mode Fiber (SMF)

Very small core, single light path. High bandwidth, long distance (lasers).

Multi-Mode Fiber (MMF)

Larger core, multiple light paths. Lower bandwidth, shorter distance (LEDs).

4. a) Explain internet checksum method with an example. b) Explain simple parity check method with an example.

Error Detection Methods

a) Internet Checksum

Used in IP, UDP, TCP. Sums 16-bit segments using one's complement, then one's complements the sum.

Seg 1: 0101010101010101

Seg 2: 0011001100110011

Sum: 1000011001001000

Checksum (1's comp): 0111100110110111

Receiver: Sum all including checksum. Result should be all 1s (0 after 1's comp).

b) Simple Parity Check

Adds a single bit to make total '1's either even or odd.

Data: 1010011 (4 ones)

Even Parity: 10100110 (4 ones)

Odd Parity: 10100111 (5 ones)

Receiver: Counts '1's. Detects odd number of errors only.

5. Analyse the types of CSMA protocol used in medium access control.

CSMA Protocol Types

Carrier Sense Multiple Access: Listen before transmit to reduce collisions.

1-Persistent CSMA

If idle, transmit immediately. If busy, continuously listen, then transmit immediately.

High collision probability under heavy load.

Non-Persistent CSMA

If idle, transmit immediately. If busy, wait random time, then re-sense.

Reduces collisions, but lower utilization.

p-Persistent CSMA

For slotted channels. Transmit with probability 'p' if idle, defer with '1-p'.

Balance utilization and collision.

CSMA/CD (Collision Detection)

Sense before transmit, and detect collisions during transmit. (Wired Ethernet)

Aborts colliding transmissions early.

CSMA/CA (Collision Avoidance)

Sense before transmit, avoid collisions (e.g., random back-off, ACKs, RTS/CTS). (Wireless Wi-Fi)

Essential for wireless where CD is hard.

6. Explain in detail the different classes of IPv4 addressing.

IPv4 Addressing Classes

32-bit addresses divided into Network ID and Host ID (Classful Addressing).

Class A

Format: 0xxxxxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx

Networks: 128 ($2^7$)

Hosts/Net: ~16.7M ($2^{24}-2$)

For very large networks. Default Mask: 255.0.0.0

Class B

Format: 10xxxxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx

Networks: 16,384 ($2^{14}$)

Hosts/Net: 65,534 ($2^{16}-2$)

For medium to large networks. Default Mask: 255.255.0.0

Class C

Format: 110xxxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx

Networks: ~2M ($2^{21}$)

Hosts/Net: 254 ($2^8-2$)

For small networks. Default Mask: 255.255.255.0

Class D (Multicast)

Format: 1110xxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx

Range: 224.0.0.0 to 239.255.255.255

For group communication.

Class E (Experimental)

Format: 1111xxxx.xxxxxxxx.xxxxxxxx.xxxxxxxx

Range: 240.0.0.0 to 255.255.255.255

Reserved for future/experimental use.

Limitations:

  • Address Depletion (inefficient use).
  • Lack of Flexibility (rigid boundaries).
  • Routing Table Bloat.

Led to CIDR (Classless Inter-Domain Routing).

7. Explain bit-oriented framing and the need for bit stuffing with an example.

Bit-Oriented Framing & Bit Stuffing

Delimits frames using bit patterns; stuffing prevents flag misinterpretation.

Bit-Oriented Framing

Uses specific bit patterns (e.g., 01111110) as frame flags.

Allows any bit pattern in data field.

Flag | Data | FCS | Flag

Need for Bit Stuffing

Prevents data from being mistaken for a flag.

If 01111110 appears in data, it breaks framing.

Bit Stuffing Example

Rule: After 5 consecutive '1's, insert a '0'.

Original Data: 0110111111011111100

Sender Stuffing:

011011111010111110100

Stuffed Payload: 011011111010111110100

Receiver Destuffing: Removes '0' after 5 '1's.
