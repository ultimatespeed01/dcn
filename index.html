7. Data word to be transmitted by the sender is 1001101. Assuming the even parity, find out the code word that will be sent to the receiver using Hamming code.

Hamming Code (Even Parity)

Data Word: 1001101 (7 bits)

1. Determine Parity Bits (r)

Formula: $2^r \ge m + r + 1$

For m=7, r=4 (since $2^4 = 16 \ge 7+4+1=12$)

Need 4 Parity Bits

2. Bit Positions (Total 11 bits)

Parity at $2^n$ positions: 1, 2, 4, 8

Data ($D_1$ to $D_7$) fill others.

C11 C10 C9 C8 C7 C6 C5 C4 C3 C2 C1

D7 D6 D5 P8 D4 D3 D2 P4 D1 P2 P1

3. Calculate Parity Bits (Even Parity)

Data Word: 1001101

P1 (1,3,5,7,9,11): 1,0,1,1,1 (4 ones) $\implies P_1=0$

P2 (2,3,6,7,10,11): 1,0,1,0,1 (3 ones) $\implies P_2=1$

P4 (4,5,6,7): 0,0,1 (1 one) $\implies P_4=1$

P8 (8,9,10,11): 1,0,1 (2 ones) $\implies P_8=0$

4. Final Codeword

10101001110

8. Explain the different types of frames exchanged during data transmission in HDLC protocol.

HDLC Frame Types

Bit-oriented protocol using specific frames for data and control.

Information Frames (I-frames)

Transport user data; carry flow/error control (piggybacking).

Includes $N_S$ (sent) and $N_R$ (received) sequence numbers.

Supervisory Frames (S-frames)

Flow and error control (ACKs, NAKs) when no user data.

Types: RR, RNR, REJ, SREJ.

Unnumbered Frames (U-frames)

Link management (setup/teardown, error reporting not data related).

No sequence numbers. e.g., SABM, DISC, UA.

14. Explain OSI Model Architecture.

OSI Model Architecture

A seven-layer conceptual model for network communication.

7. Application Layer

Function: Provides network services directly to end-user applications. Deals with user interaction and application-specific protocols (e.g., HTTP, FTP, DNS, SMTP).

6. Presentation Layer

Function: Handles data format translation, encryption, and compression. Ensures data is in a readable format for the application layer (e.g., JPEG, MPEG, ASCII, EBCDIC).

5. Session Layer

Function: Establishes, manages, and terminates communication sessions between applications. Handles dialogue control and synchronization (e.g., NetBIOS, RPC).

4. Transport Layer

Function: Provides reliable or unreliable end-to-end data delivery. Handles segmentation, reassembly, flow control, and error control (e.g., TCP, UDP).

3. Network Layer

Function: Manages logical addressing (IP addresses) and routing of packets across different networks. Handles fragmentation and logical path determination (e.g., IP, ICMP, ARP).

2. Data Link Layer

Function: Provides reliable data transfer across a physical link. Handles physical addressing (MAC addresses), framing, error detection, and flow control on the local network (e.g., Ethernet, PPP, HDLC).

1. Physical Layer

Function: Defines the physical characteristics of the network medium and how bits are transmitted. Deals with cables, connectors, voltage levels, and data rates (e.g., coaxial cable, fiber optics, RJ45).

Section C

1. Explain TCP/IP Model Architecture.

TCP/IP Model Architecture

A four-layer hierarchical protocol suite that forms the basis of the internet.

4. Application Layer

Function: Provides network services directly to end-user applications. Handles data formatting, encryption, and compression at the application level.

3. Transport Layer

Function: Manages end-to-end communication between applications. TCP provides reliable, connection-oriented data transfer (segmentation, flow control, error control). UDP provides unreliable, connectionless data transfer.

2. Internet Layer

Function: Responsible for logical addressing (IP addresses), routing packets across different networks, and fragmentation/reassembly of packets. It's connectionless and provides best-effort delivery.

1. Network Access Layer

Function: Defines how data is physically transmitted over the network medium. Handles physical addressing (MAC addresses), framing of data into bits, and error detection on the local link.

3. Explain the Transport Layer protocols of TCP/IP model.

TCP/IP Transport Layer Protocols

Manages end-to-end communication between applications.

TCP (Transmission Control Protocol)

Reliable, Connection-Oriented: Ensures data delivery, order, and integrity. Handles segmentation, flow control, error control (ACKs, retransmissions).

  • Three-way handshake
  • Congestion control
  • Used for web (HTTP), email (SMTP), file transfer (FTP).

UDP (User Datagram Protocol)

Unreliable, Connectionless: Faster, lower overhead. No guarantees of delivery, order, or error checking beyond checksum.

  • No handshake
  • No congestion control
  • Used for streaming video/audio, DNS, VoIP.

4. Explain the Internet Layer protocols of TCP/IP model.

TCP/IP Internet Layer Protocols

Responsible for logical addressing, routing, and fragmentation of data packets across networks.

IP (Internet Protocol)

The primary protocol for logical addressing (IPv4 and IPv6) and routing packets across different networks. It's connectionless and provides best-effort delivery.

ARP (Address Resolution Protocol)

Maps IP addresses (logical) to physical MAC addresses on a local network segment, enabling communication within the same network.

RARP (Reverse ARP)

Allows a device to discover its own IP address from its MAC address, typically used by diskless workstations during boot-up (largely superseded by DHCP).

ICMP (Internet Control Message Protocol)

Used by network devices to send error messages and operational information, such as destination unreachable or time exceeded, and for diagnostic tools like ping and traceroute.

IGMP (Internet Group Management Protocol)

Manages multicast group memberships, allowing devices to join or leave multicast groups for efficient delivery of data to multiple recipients (e.g., streaming video).

6. Explain distance vector routing algorithm with an example.

Distance Vector Routing Algorithm

Routers share their routing tables with directly connected neighbors. Iteratively updates paths based on shortest distance.

Concept: "Routing by rumor"

Each router maintains a distance vector (table) of shortest distances to all other networks and the next hop for each. It periodically sends this table to its neighbors. Neighbors then update their own tables if they find a shorter path through the sender.

Example:

A

Cost to B: 1

B

Cost to C: 2

C

Router A learns about C via B. If A's cost to B is 1, and B's cost to C is 2, A calculates its cost to C as 1+2=3. If A later hears a cheaper path to C from another neighbor, it updates its table.

Drawback: "Count-to-infinity" problem and slow convergence.

7. What is line coding? Explain the different schemes of line coding for the data stream 10011101.

Line Coding

Process of converting digital data (binary bits) into digital signals for transmission over a physical medium.

Data Stream: 10011101

1. Unipolar NRZ

0V for '0', positive voltage for '1'.

2. Polar NRZ-L (Non-Return to Zero - Level)

Positive voltage for '0', negative voltage for '1'.

3. Polar NRZ-I (Non-Return to Zero - Invert)

Inversion on '1', no change on '0'. (Assume start high)

4. Manchester

Transition in the middle of each bit period. '0' (high to low), '1' (low to high).

8. Discuss in detail the design of Fiber Optic cables and how it transmits data? Explain its types.

Fiber Optic Cables

Transmits data as pulses of light through thin glass or plastic fibers.

Design:

  • Core: Innermost part, made of glass or plastic, where light travels.
  • Cladding: Surrounds the core, with a lower refractive index to reflect light back into the core (Total Internal Reflection).
  • Buffer Coating: Protects the fiber from moisture and physical damage.
  • Outer Jacket: Provides overall protection to the cable.

Data Transmission:

Electrical signals are converted into light pulses by an LED or laser diode. These light pulses travel through the fiber core, reflecting off the cladding due to total internal reflection. At the receiver, a photodiode converts the light pulses back into electrical signals.

Types:

Single-Mode Fiber (SMF)

Small core (approx. 9 µm), allows only one light path. Used for long distances and high bandwidth.

Multi-Mode Fiber (MMF)

Larger core (50 or 62.5 µm), allows multiple light paths. Used for shorter distances.

9. a) Explain internet checksum method with an example. b) Explain simple parity check method with an example.

Error Detection Methods

a) Internet Checksum

Used in TCP, UDP, IP, ICMP to detect errors in headers. Sums 16-bit words, then takes one's complement of the sum.

Example: Data: 10101010 01010101
  • Add: 10101010 + 01010101 = 11111111
  • One's Complement: 00000000 (Checksum)
  • Transmitted: 10101010 01010101 00000000
  • Receiver adds all: If sum is all 1s, no error.

b) Simple Parity Check

Adds a single parity bit to ensure an even or odd number of 1s in the data.

Example: Data: 1010101 (4 ones)
  • Even Parity: Add '0' (total 4 ones). Transmitted: 10101010
  • Odd Parity: Add '1' (total 5 ones). Transmitted: 10101011
  • Receiver checks parity. Detects single-bit errors.

10. Analyse the types of CSMA protocol used in medium access control.

CSMA Protocols (Carrier Sense Multiple Access)

Protocols where nodes listen to the shared transmission medium before transmitting to avoid collisions.

1-Persistent CSMA

If channel is idle, transmit immediately. If busy, continuously sense and transmit with probability 1 when idle.

High collision probability under heavy load.

Non-Persistent CSMA

If channel is idle, transmit. If busy, wait a random amount of time, then sense again.

Reduces collisions but increases delay.

p-Persistent CSMA

If channel is idle, transmit with probability 'p'. With probability '1-p', wait for next time slot and repeat.

Balances collision and delay. Used in Wi-Fi.

CSMA/CD (Collision Detection)

If collision detected during transmission, abort and retransmit after a random back-off time. Used in wired Ethernet.

CSMA/CA (Collision Avoidance)

Attempts to avoid collisions before they happen (e.g., by using RTS/CTS handshake). Used in wireless networks (Wi-Fi).

12. Explain bit-oriented framing and the need for bit stuffing with an example.

Bit-Oriented Framing & Bit Stuffing

Methods used in data link layer protocols (like HDLC) to delineate frames and prevent flag sequences from appearing in data.

Bit-Oriented Framing:

Uses a specific bit pattern, the "flag" ($01111110$), to mark the beginning and end of a frame. This allows the receiver to identify where one frame ends and the next begins.

Need for Bit Stuffing:

If the actual data payload contains the flag sequence ($01111110$), the receiver might misinterpret it as an end-of-frame marker, leading to frame truncation and data loss. Bit stuffing prevents this ambiguity.

Bit Stuffing Mechanism:

Whenever the sender encounters five consecutive '1's in the data stream, it automatically inserts a '0' bit. This ensures that the flag sequence cannot accidentally appear within the data. The receiver performs "bit destuffing" by removing any '0' that follows five consecutive '1's.

Example:

Original Data: 01101111110100

Flag: 01111110

Stuffed Data (Sender):

011011111010100

After five 1s, a 0 is stuffed.

Transmitted Frame:

01111110 011011111010100 01111110

The stuffed '0' ensures the data doesn't mimic the flag.
