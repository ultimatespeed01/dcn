Fundamentals of Data Communication

This section delves into the foundational aspects of how data is exchanged, including the basic modes of flow, essential components involved, and the nature of signals used in communication.

Data Flow Modes

Understanding how data travels between devices is crucial. Our report highlights three primary modes:

Simplex

Sender → Receiver

One-way traffic (e.g., TV broadcast).

Half-Duplex

Device A ⇆ Device B

Two-way, but not simultaneous (e.g., walkie-talkies).

Full-Duplex

Device A ⇄ Device B

Two-way, simultaneous (e.g., phone call).

Core Components of Data Communication

  • Message: The information (data) to be communicated.
  • Sender: The device that sends the data message.
  • Receiver: The device that receives the data message.
  • Transmission Medium: The physical path for message travel.
  • Protocol: A set of rules governing data communication.

Analog vs. Digital Signals

Analog Signals

Continuous waveforms with infinite values over a range. Represent info by varying amplitude/frequency (e.g., human voice).

Digital Signals

Discrete-time signals with finite values (0s and 1s). Example: data in computers.

Data Integrity & Error Management

Ensuring that data arrives at its destination without corruption is a critical aspect of networking. This involves techniques for both detecting and, in some cases, correcting errors that occur during transmission.

Error Detection vs. Correction

Error Detection

Identifies if errors occurred. Receiver knows data is corrupt but can't fix it. Examples: Parity Check, CRC.

Error Correction

Detects, locates, and corrects errors. More complex. Example: Hamming code.

Parity Check Example (Data: 1010011)

Data word 1010011 has 4 ones (even).

  • Even Parity: Parity bit is 0. Codeword: 10100110
  • Odd Parity: Parity bit is 1. Codeword: 10100111

Internet Checksum Method

A common error detection method for protocol headers (e.g., IP, TCP). It involves summing 16-bit words of the header (with end-around carry for overflows) and then taking the one's complement of the final sum. The receiver performs the same sum including the received checksum; if the result is all 1s (0xFFFF), no error is detected.

While simple and fast, it's not foolproof and can miss certain error patterns.

Channel Utilization & Access

Efficiently sharing communication channels among multiple users is key. This section explores multiplexing techniques and random access protocols like Aloha.

Aloha Protocol Throughput

The Aloha protocols are foundational random access methods. Their theoretical maximum throughputs highlight the impact of collision management strategies.

Time Division Multiplexing (TDM)

TDM divides channel time into slots for different users. Key types include:

Synchronous TDM (STDM)

Fixed slots per user, regardless of data. Can be inefficient if users are idle.

Frame: [A1 | B_empty | C1]

Asynchronous TDM (ATDM)

Dynamic slots, only for active users. More efficient for bursty traffic.

Frame: [A1 | C1 | A2]

Frequency Division Multiplexing (FDM) & Guard Bands

FDM assigns different frequency bands to users. Guard bands are unused frequency strips between channels to prevent interference or crosstalk, ensuring signal clarity.

Network Protocols & Switching

This section explores fundamental switching techniques that form the backbone of network communication, alongside key data link layer protocols like HDLC and authentication methods in PPP.

Circuit Switching vs. Packet Switching

Circuit Switching

  • Dedicated physical path established.
  • Resources reserved for the duration.
  • Example: Traditional telephone network.

Packet Switching

  • Data broken into packets.
  • Packets routed independently.
  • Resources shared dynamically.
  • Example: Internet.

HDLC Frame Types

High-level Data Link Control (HDLC) uses distinct frame types for different communication purposes:

  • I-Frames (Information): Carry user data and control info (sequence numbers, ACKs).
  • S-Frames (Supervisory): For flow and error control (e.g., ACK, NAK, ready/not ready). Do not carry user data.
  • U-Frames (Unnumbered): For link management (setup, disconnect, error reporting).

The Control field within the HDLC frame header distinguishes these types.

PPP Authentication: PAP vs. CHAP

PAP (Password Authentication Protocol)

  • Sends username/password in clear text.
  • Vulnerable to eavesdropping.
  • Two-way handshake.

CHAP (Challenge-Handshake Auth. Protocol)

  • More secure; password not sent directly.
  • Uses a three-way challenge-response.
  • Periodically re-challenges.

Routing Principles & Congestion Control

Efficiently directing data packets through a network and managing traffic to prevent overloads are crucial for network performance. This section covers routing table strategies and congestion control mechanisms.

Static vs. Dynamic Routing Tables

Static Routing

Routes manually configured. Simple, secure, less overhead. Not adaptive to network changes.

Dynamic Routing

Routes learned automatically via protocols (RIP, OSPF). Adapts to changes. More complex and overhead.

Distance Vector Routing Algorithm

An iterative algorithm where routers share their distance vectors (cost to destinations) with neighbors. Each router updates its table based on neighbors' information to find the shortest path. Prone to slow convergence (count-to-infinity problem).

Key principle: Bellman-Ford algorithm. Solutions like split horizon mitigate issues.

Congestion Control: Leaky Bucket vs. Token Bucket

These algorithms shape traffic to prevent network congestion.

Leaky Bucket Algorithm

  • Output: Smooth, constant rate.
  • Mechanism: Data queues and "leaks" out.
  • Bursts: Absorbs small bursts; large bursts cause packet loss if buffer overflows.
  • Idle Periods: No credit saved.
  • Congestion Aspect: Prevents by smoothing traffic to a strict rate.

Token Bucket Algorithm

  • Output: Can be bursty (up to token capacity), then average rate.
  • Mechanism: Packets need tokens (generated at a constant rate) to transmit.
  • Bursts: Allows by consuming accumulated tokens.
  • Idle Periods: Tokens accumulate, allowing larger bursts later.
  • Congestion Aspect: Prevents by policing/shaping average rate and burst size. More flexible.

Addressing & Network Structures

This section explores how devices are uniquely identified using IPv4 addressing schemes and how networks are physically or logically arranged through various topologies.

IPv4 Classful Addressing Overview

Traditional IPv4 addressing divided the space into classes, each with a different allocation for network and host IDs. While largely superseded by CIDR, understanding classes provides historical context.

Class A

(1-126.x.x.x)

Few Networks, Many Hosts

Class B

(128-191.x.x.x)

Moderate Networks & Hosts

Class C

(192-223.x.x.x)

Many Networks, Few Hosts

Network Topologies: Ring vs. Mesh

Ring Topology

A ↔ B ↔ C ↔ D ↔ (A)

Devices in a circle. Orderly access. Single point of failure can disrupt.

Mesh Topology (Conceptual)

A B C D

High reliability (many interconnections). Full mesh is expensive.

Data Encoding & Physical Transmission

This section covers how binary data is converted into signals for transmission and the characteristics of transmission media like fiber optics, including mechanisms like bit stuffing for data transparency.

Line Coding Schemes (Conceptual for data 1001)

Line coding converts binary data to digital signals. Different schemes offer trade-offs in synchronization, DC component, and bandwidth.

Polar NRZ-L (1: +V, 0: -V)

Manchester (1: L->H, 0: H->L mid-bit)

Bit-Oriented Framing & Bit Stuffing

Bit-oriented protocols use flags (e.g., 01111110) to mark frame boundaries. Bit stuffing is needed for data transparency: if the flag pattern appears in data, the sender inserts an extra '0' after five consecutive '1's. The receiver removes this stuffed '0'.

Example: Data ...0111111... becomes ...01111101... after stuffing.

Fiber Optic Cable Design

Fiber optic cables transmit data as light pulses using Total Internal Reflection (TIR). Key components include:

  • Core: Inner light-carrying part (high refractive index).
  • Cladding: Surrounds core (lower refractive index, enables TIR).
  • Buffer Coating: Protective plastic layer.
  • Strength Members: Provide tensile strength (e.g., Aramid yarn).
  • Outer Jacket: External protection.

Types include Single-Mode (SMF) for long-haul, high bandwidth, and Multi-Mode (MMF) for shorter distances.

Advanced Access Control & Layered Protocols

This section explores more sophisticated medium access control mechanisms like CSMA and its variants, and key protocols within the TCP/IP network layer that manage addressing and routing.

CSMA (Carrier Sense Multiple Access) Protocols

CSMA protocols involve stations listening to the channel before transmitting to reduce collisions.

1-Persistent CSMA

If channel idle, transmit. If busy, sense continuously and transmit immediately when idle. Can be greedy, high collision risk if multiple stations wait.

Non-Persistent CSMA

If channel idle, transmit. If busy, wait random time then sense again (doesn't sense continuously). Reduces collisions but can increase delay.

p-Persistent CSMA

If channel idle, transmit with probability 'p', or defer with (1-p) to next slot. Balances collision reduction and idle time. Used in slotted channels.

CSMA/Collision Detection (CD)

"Listen while talk." If collision detected during transmission, abort, send jam signal, backoff randomly. Used in wired Ethernet.

CSMA/Collision Avoidance (CA)

Tries to *avoid* collisions. Uses IFS, random backoff, ACKs. Optional RTS/CTS. Used in wireless (Wi-Fi).

TCP/IP Network Layer Protocols

The Network (Internet) Layer handles logical addressing and routing.

  • IP (Internet Protocol): Primary protocol. Connectionless, unreliable datagram delivery. Handles addressing (IPv4/IPv6) and routing.
  • ICMP (Internet Control Message Protocol): Works with IP for error reporting and diagnostics (e.g., ping, destination unreachable).
  • Routing Protocols (e.g., RIP, OSPF, BGP): Enable routers to build and maintain routing tables for IP to use.
